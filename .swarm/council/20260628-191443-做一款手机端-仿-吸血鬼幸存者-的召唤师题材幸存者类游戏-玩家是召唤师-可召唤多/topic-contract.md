## Objective(目标)
做一款手机端、仿《吸血鬼幸存者》的召唤师题材幸存者类游戏：玩家是召唤师，可召唤多种宠物自动作战、宠物之间能配合连携、宠物可随局内/局外成长升级

## Scope(范围)
- in:
  - 一关可玩的核心循环：角色（召唤师）移动 + 怪潮不断刷新 + 自动战斗 + 局内计时/难度递增
  - 召唤系统：可召唤多种宠物（至少 3-5 种原型），宠物自动寻敌作战，体现召唤师"指挥而非亲自砍"的特色
  - 宠物配合/连携机制：宠物之间能产生 1+ 种协同效果（具体形式由方案设计）
  - 宠物成长升级：局内拾取经验/掉落升级 + 类《吸血鬼幸存者》的升级三选一卡片；可选的局外永久成长雏形
  - 移动端触屏操作（虚拟摇杆/拖拽/点击），竖屏或横屏其一，HUD + 选卡 + 结算基础 UI
  - 本地存档（localStorage 级别即可）
- out:
  - 完整商业化（内购、广告、账号体系、服务器存档）
  - 海量内容（几十种宠物/多关卡/剧情）、多人联机、排行榜后端
  - 复杂美术管线；demo 阶段用免费/开源/AI 生成素材或几何占位图即可

## Hard constraints(硬约束)
- budget / 成本: 极低，接近 0；优先免费开源框架与免费/AI 生成美术音效
- stack / 技术栈: **H5 / Web，移动浏览器优先**；代码将交由顶级 AI 生成，故技术栈必须主流、生态成熟、AI 训练充分、易于 AI 持续迭代；**TypeScript 优先**，倾向成熟 2D 框架（如 Phaser 3 / PixiJS）或轻量 Canvas，避免冷门技术；需在中端手机浏览器上跑到可玩帧率（大量怪物的性能是关键）
- deadline / 时限: 几周内产出"能上手玩"的原型 Demo（核心循环跑通优先于内容量）
- must-keep / 必须保留: 三大核心不可删——①召唤多种宠物自动作战 ②宠物之间能配合连携 ③宠物可成长升级；保留《吸血鬼幸存者》式自动攻击 + 割草潮 + 升级三选一的爽感；手机端可玩

## Scoring rubric(评分维度,tender)
- 契合目标（是否真正实现"召唤师 + 宠物配合 + 成长"三核心，且像吸血鬼幸存者）
- 可行性（几周内、由 AI 生成代码能否落地）
- 成本（时间 / 工时 / 美术资源 / 维护）
- 风险（性能、范围蔓延、玩法不好玩、技术坑）
- 可维护性（架构是否清晰、对 AI 持续迭代是否友好）
- 趣味性·手感（割草爽感 + 召唤师指挥的独特乐趣，domain 维度）
- 移动端适配（触屏操作与性能，domain 维度）
