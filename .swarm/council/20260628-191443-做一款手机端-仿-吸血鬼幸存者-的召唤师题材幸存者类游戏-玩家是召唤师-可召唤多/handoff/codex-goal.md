# Codex 目标模式 — 召唤师幸存者 H5（直接复制下面整段作为目标）

---

你的目标：从零构建一款仿《吸血鬼幸存者》的"召唤师"题材 **H5 手机网页**割草游戏 Demo，4 周内做到手机浏览器可玩、不卡、可外发。

**开工前必读、全程遵守：**
- 完整方案：`.swarm/council/20260628-191443-做一款手机端-仿-吸血鬼幸存者-的召唤师题材幸存者类游戏-玩家是召唤师-可召唤多/plan.md`
- 工程约束：项目根 `AGENTS.md`

**技术栈（固定）**：Phaser 3 + TypeScript(strict) + Vite。无后端，localStorage 存档，jsfxr 音效。

**游戏核心（必保三项）**：①召唤 5 种宠物（剑齿狼/焰魔/霜灵蝶/雷羽鹰/治愈花）自动作战；②宠物连携 = 元素反应(火+冰融化、冰+雷超导) + 一层薄"猎印"指挥；③宠物成长 Lv1-5 + 升级三选一 + 轻度局外魂晶成长树。

**操作**：竖屏 + 全屏任意拖拽（摇杆为设置备选）。

**美术**：你调用图像模型生成**全套暗色哥特复古像素素材**，按 AGENTS.md 的固定 prompt 模板 + 强制后处理（降采样/调色板量化/抠边）打成纹理图集，体积 ≤0.8MB；动画用程序化 tween，不做逐帧。

**执行方式（重要）**：按 **M0→M5 里程碑顺序长跑**。每个里程碑结束必须 `npm run dev` 能运行并通过验收命令（`npx tsc --noEmit` / `npm run build` / `npm run test`）才进下一个。先做能跑的最小闭环再加内容。**M1 必须先实现 SpatialGrid + 对象池并通过"200 怪 ≥30fps"压测，不达标不进 M2。** 每个里程碑完成后用中文 git 提交。

**里程碑：**
- **M0 初始化**：脚手架 Vite+Phaser+TS + 确认 AGENTS.md + 竖屏 FIT + 全屏拖拽 + 空场景能跑 + git 提交。
- **M1 性能地基+最小闭环**：SpatialGrid + 对象池 + 200 怪压测(闸门) + 刷怪难度曲线 + 1 宠自动战斗(索敌锁定粘性+伤害100ms) + 经验升级框架 + HUD 骨架。
- **M2 召唤+连携+成长**：5 宠数据驱动 + 元素标记 + 薄猎印 + 2 连携(防雪崩，先写单测) + 三选一卡池(权重优先已有宠) + 宠物 Lv1-5。
- **M3 全套像素美术**：定 Art Bible + 图像模型生成全套素材 + 后处理 + 打图集 + 替换占位 + tint/flipX 动画。
- **M4 局外成长+完整UI**：魂晶成长树 + SaveManager(防损坏/版本迁移) + 完整 HUD/连携飘字/磁吸/受击/结算/标题。
- **M5 打磨+性能+真机+发布**：音效(jsfxr)+震动 + 视野裁剪/降级/DeviceDetect + iOS 音频解锁/手势/安全区 + 真机测试 + 数值平衡 + Service Worker + 生产构建部署静态托管(Vercel/Cloudflare/GitHub Pages)。

**性能红线**：碰撞统一 SpatialGrid + 距离判定(禁用 Arcade 全量)；全对象池；伤害 100ms tick；宠物索敌 0.2s + 锁定粘性；连携 per-target 1.5s 冷却 + 每帧触发封顶 + 扩散标记不二次引爆。

**完成标准（done）**：三核心齐备且手机可玩；全套像素美术风格统一、图集 ≤0.8MB；中端 150-200 怪 ≥30fps、低端降级 ≥24fps；局外存档刷新保留、坏档不崩；`tsc` 零报错 + `vite build` < 1.5MB + 连携有单测 + 可外发链接。
