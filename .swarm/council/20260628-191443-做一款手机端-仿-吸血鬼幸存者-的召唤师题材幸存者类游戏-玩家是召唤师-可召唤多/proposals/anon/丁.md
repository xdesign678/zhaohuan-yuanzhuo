## 一句话方案
采用 Phaser 3 (WebGL) + TypeScript 构建的高性能 H5 游戏，以轻量 ECS 架构驱动海量实体，主打"宠物全自动战斗 + 元素连携反应 + 三选一卡池"的生存割草体验。

## 推荐方案
**技术栈选择**
*   **框架与渲染**：选用 **Phaser 3** 配合 **TypeScript**。Phaser 3 是当前极其成熟的 H5 2D 游戏框架，默认使用 WebGL 批处理渲染，API 极其丰富且网上 AI 训练数据覆盖率高，AI 生成代码的准确率和可运行率有保障。
*   **语言**：TypeScript 的强类型有助于在持续迭代和增加多种宠物/卡牌时减少低级拼写和类型 Bug，便于 AI 维护项目结构。

**整体代码架构**
*   **核心架构**：基于轻量级的 **ECS (Entity-Component-System)** 思想。
*   **关键模块**：
    *   `GameScene`：主循环、UI 层与游戏层的解耦管理。
    *   `EntityManager` / `PoolManager`：统一管理主角、宠物、怪物、弹幕、掉落物实体的生命周期与对象池分配。
    *   `DirectorSystem` (AI 导演)：根据存活时间控制敌人波次刷新、怪物密度与属性强度。
    *   `CombatSystem`：处理索敌策略、物理交集、伤害结算与元素连携判定。
    *   `UpgradeSystem`：管理经验值拾取、升级逻辑、"三选一"卡池随机抽取与应用。

**3-5 种原型宠物设计**
1.  **火球灵 (Fire Sprite)**：【远程/单体/火】周期性向最近敌人发射火球，单体伤害高。
2.  **毒蘑菇 (Toxic Shroom)**：【AOE/场地/毒】在自身周围散播毒雾，进入区域的敌人减速并持续掉血（附加中毒 Buff）。
3.  **旋风兽 (Gale Beast)**：【近战/生存/风】环绕在召唤师周围快速旋转，造成轻微伤害并击退靠近的敌人，为主角提供安全区。
4.  **雷电鸟 (Thunderbird)**：【远程/弹射/雷】释放闪电链，在密集敌人间弹射，弹射伤害递减，适合清杂兵。

**宠物配合/连携机制**
*   **设计核心**：通过“状态异常(Buff) + 特定元素攻击”触发进阶反应。
*   **具体连携例子**：
    1.  **毒爆 (毒 + 火)**：当敌人身上带有"中毒"状态（毒蘑菇赋予）时，被"火球灵"击中，会引发爆炸，造成目标最大生命值百分比的范围真实伤害，并清除中毒状态。
    2.  **感电扩散 (风 + 雷)**：被"旋风兽"击退的敌人会被短暂挂上"风破"易伤标记，此时若被"雷电鸟"击中，闪电链的弹射范围翻倍，且不再衰减伤害。
*   **实现方式**：在 `CombatSystem` 的伤害结算环节，读取目标实体的 `BuffList`。若受到攻击属性匹配连携条件，则取消普通伤害逻辑，触发特殊效果（抛出 `ComboEvent`，生成新的范围伤害实体）。

**成长升级系统**
*   **局内升级**：
    *   敌人死亡掉落经验宝石，召唤师靠近触发吸附拾取。经验条满后暂停游戏时间，弹出"三选一"卡牌 UI。
    *   **卡池示例**：【解锁】获得新宠物"雷电鸟"；【升级】"火球灵"弹道数量+1；【被动】全体宠物冷却时间-10%；【强化】"毒蘑菇"毒雾范围+20%。
*   **局外成长 (雏形)**：利用 H5 `localStorage` 记录金币，可在主菜单花费金币提升召唤师初始移动速度、经验获取倍率。

**移动端操作方案与 UI 布局**
*   **操作**：全屏任意位置滑动（虚拟摇杆隐藏显示）控制召唤师移动，纯单指操作。宠物全自动运行。
*   **UI 布局**：
    *   顶部：经验条（贯穿全宽）、居中显示存活倒计时。
    *   左上角：玩家血条。右上角：暂停/设置按钮。
    *   底部：小图标排列展示当前已获得的宠物及等级（Max 5级）。
    *   升级时：屏幕中央弹出 3 张竖向排列或并排的大卡片，点击即选。

**性能方案（保障海量怪物不卡顿）**
1.  **对象池 (Object Pool)**：怪物、弹幕、伤害飘字、经验宝石 100% 使用对象池，杜绝 `new` 操作造成的内存抖动与 GC 停顿。
2.  **空间哈希网格 (Spatial Hash Grid)**：将屏幕按大小划分为网格。寻敌逻辑与碰撞检测只遍历相邻网格内的实体，从 $O(N^2)$ 降为 $O(N)$，解决几百只怪物同屏时的 CPU 瓶颈。
3.  **精简物理**：禁用 Phaser 默认的 Arcade 物理碰撞反弹，仅保留简单的 AABB（矩形）重叠检测。允许怪物模型重叠，避免复杂的物理挤压计算。
4.  **计算降频**：每隔 3-5 帧才进行一次索敌和 AI 移动方向更新，不需要 60 帧每秒的高频计算。

## 关键决策与取舍
*   **选择纯渲染框架而非 DOM/前端框架**：放弃用 React/Vue 渲染实体的思路。割草游戏核心在于成百上千对象的坐标高频更新，必须用 Phaser (WebGL) 直接进行底层批处理渲染。
*   **剥离物理引擎，自定义简单交集**：传统的重度物理引擎（涉及动量、挤压）对割草游戏是灾难。我们选择“允许穿模叠加”，只需判断矩形相交即扣血，极大保全了手机端的 CPU 资源。
*   **召唤师零攻击**：为强化“召唤”特色，主角没有任何攻击能力，完全依赖走位和宠物护航，从机制上与传统射击肉鸽区隔。

## 步骤 / 任务分解
*   **W1：核心循环与性能地基**。
    *   搭建 Phaser + TypeScript 框架。实现虚拟摇杆与主角移动。
    *   构建核心对象池、简单碰撞网格。
    *   实现无限出怪逻辑、火球灵单体自动索敌打怪、怪物掉落经验。
*   **W2：内容填充与卡池系统**。
    *   实现经验拾取、升级判定、暂停及“三选一”抽卡 UI。
    *   加入毒蘑菇、旋风兽、雷电鸟的攻击逻辑与对应的升级卡牌。
*   **W3：连携系统与打磨交付**。
    *   加入 Buff 系统和“毒爆”等连携反应判断。
    *   完善 UI（血条、伤害飘字）、加入游戏结束与结算、利用免费素材拼装可用版本。跑真机性能测试。

## 风险与缓解
*   **风险 1：千怪同屏导致的手机浏览器内存崩溃或掉帧过热。**
    *   *缓解*：设置“同屏实体硬上限”（如 300 个）。若超限，停止刷怪，改为提升场上现有怪物的血量、速度和攻击力（即“怪物激怒”机制），保证压力不减但渲染开销锁死。
*   **风险 2：AI 生成代码在对象池和网格索引上容易犯低级错误（如对象回收未重置状态）。**
    *   *缓解*：在 Prompt 中明确提供对象池 Lifecycle（`spawn`, `despawn`, `reset`）的样板约束，要求 AI 在复用前必须清空实体的状态残余。

## 成本 / 工时估计
*   **低成本**：仅涉及工程代码，美术用几何图形或免费开源库 (Kenney) 即可，音频用 JSfxr 生成合成音。
*   **最耗时/易翻车点**：**高性能碰撞与索敌机制的实现**。AI 很容易写出导致性能崩盘的双层循环，这一块的架构设计必须非常严密，预计占总工时的 30-40%。

## 验收标准 (done-when，可检验)
1.  在 iPhone 11 / 骁龙 870 等中端手机自带浏览器上，滑动摇杆流畅，同屏 300 怪时不掉出 45 fps。
2.  杀怪能爆经验宝石，主角靠近能吸附拾取，经验满触发升级，弹出 3 张随机卡牌。
3.  选择卡牌后，宠物/属性立即获得更新。
4.  至少一种连携效果（如毒蘑菇+火球灵触发大范围真实伤害）能稳定触发并看到伤害提升。

## 我放弃的备选
我放弃了采用 Unity WebGL 或者 Cocos Creator，因为它们虽然做游戏专业，但在“完全依赖纯文本 AI 生成代码并快速迭代”这条路上，它们的生态绑定较深（依赖编辑器拖拽或生成巨大的构建文件），不便于大语言模型直接输出轻量级的纯代码工程并独立运行。Phaser 3 纯代码驱动的特质与 AI Agent 的结合更为自然、高效。
